Code for America:
Re-imagine 911 Data Exploration Dashboard Story

Note: Dashboard size is a little big so some scrolling is necessary to view all content. For better experience, please view in full screen.


Project background: The above data exploration effort serves as a prototype for using data to uncover patterns and opportunities to improve the current 911 system. Under the guidance of my manager (in terms of project requirements, background knowledge, and thinking points), I spearheaded the above analysis effort using SQL and Tableau to explore open 911 data collected by Code for America and answer questions that stakeholders might be immediately interested in.

Context: According to research by the Vera Institute of Justice, most calls received by 911 call centers are non-urgent or not regarding crime actively happening: 911 has become the immediate resource the public turns to when they meet situations that require assistance. However, for certain issues/incidents people call for, the police might not be the best suited for handling such cases.

Hence, in this project, we focus on looking into opportunities to divert calls to non-police responders for better public service and case resolutions. In the above story, we explain the project's context, review the effectiveness of project data standardization efforts, and demonstrate exploration and analysis that can be performed using data to uncover insights to divert 911 calls to non-police responders and improve the current 911 system.
